DOCUMENTAÇÃO DO SISTEMA – CLÍNICA VETERINÁRIA

1. VISÃO GERAL

O sistema Clínica Veterinária foi desenvolvido para auxiliar no gerenciamento completo de uma clínica veterinária,
permitindo o controle de clientes, animais, consultas e utilizadores.

Funcionalidades principais:
- Cadastro e gestão de clientes
- Registro de animais
- Agendamento e controle de consultas veterinárias
- Gestão de utilizadores (admin, staff, cliente)
- Área do cliente para visualização de dados
- Operações CRUD completas

Tecnologias utilizadas:
- Backend: Python + Flask
- Base de dados: MySQL


2. ESTRUTURA DE PASTAS

O projeto segue uma estrutura organizada e modular.
Consulte o arquivo FileTree_Clinica_Veterinaria.md para detalhes completos.


3. MODELOS DE DADOS (MYSQL)

TABELA: users
- id (INT, PK): Identificador do utilizador
- username (VARCHAR): Nome de utilizador
- password (VARCHAR): Palavra-passe
- role (VARCHAR): Perfil do utilizador
- cliente_id (INT, FK): Cliente associado
- created_at (DATETIME): Data de criação

TABELA: clientes
- id (INT, PK): Identificador do cliente
- nome (VARCHAR): Nome
- email (VARCHAR): Email
- telefone (VARCHAR): Telefone
- morada (VARCHAR): Morada
- created_at (DATETIME): Data de cadastro

TABELA: animais
- id (INT, PK): Identificador do animal
- cliente_id (INT, FK): Dono do animal
- nome (VARCHAR): Nome
- especie (VARCHAR): Espécie
- raca (VARCHAR): Raça
- data_nascimento (DATE): Data de nascimento

TABELA: consultas
- id (INT, PK): Identificador da consulta
- animal_id (INT, FK): Animal atendido
- data_hora (DATETIME): Data e hora
- motivo (TEXT): Motivo da consulta
- notas (TEXT): Observações
- created_at (DATETIME): Data de criação


4. ROTAS E FUNCIONALIDADES

4.1 AUTENTICAÇÃO
/login           GET/POST   Login do utilizador
/logout          GET        Logout do sistema
/meus_dados      GET        Dados do utilizador logado

4.2 ÁREA DO CLIENTE
/minha_area                          GET
/clientes/<id>/novo_animal           GET/POST
/clientes/<id>/nova_consulta         GET/POST

4.3 GESTÃO DE CLIENTES
/tabela_clientes                     GET
/registrar_novo_cliente              GET/POST
/editar_cliente/<id>                 GET/POST
/deleta_cliente/<id>                 POST

4.4 GESTÃO DE UTILIZADORES
/tabela_utilizadores                 GET
/registrar_novo_utilizador            GET/POST
/editar_users/<id>                   GET/POST
/trocar_password/<id>                GET/POST
/deleta_utilizador/<id>              POST

4.5 GESTÃO DE ANIMAIS
/tabela_animais                      GET
/editar_animal/<id>                  GET/POST
/deleta_animal/<id>                  POST

4.6 GESTÃO DE CONSULTAS
/tabela_consultas                    GET
/editar_consulta/<id>                GET/POST
/deleta_consulta/<id>                POST


5. SEGURANÇA

- Apenas utilizadores autenticados podem acessar o sistema
- Algumas ações são restritas por role
- Senhas armazenadas em texto simples (recomendado hashing em produção)
